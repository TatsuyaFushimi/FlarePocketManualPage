# 編集マニュアル v2｜外注編集者向け

---

## 修正依頼の使い方

修正依頼は **時刻 ＋ 番号（一言）** の形式で送ります。
このマニュアルの番号を参照して対応してください。

```
例：
00:20 TE1（テロップタイミング修正お願いします）
00:35 TX1（指示代名詞を具体語に置き換えお願いします）
11:10 MO1（一般人の顔にモザイクお願いします）
```

# TE｜テロップ

---

## TE1｜表示タイミング

**ルール：** テロップは演者が話し始めた瞬間に表示・終わった瞬間に消す

**NG例：** 発話の0.5秒後にテロップが出る／話し終わった後もテロップが残る

**OK例：** 「売れました」と口が動くと同時にテロップ表示、発話終了と同時に消える

**理由：** タイミングのズレが気になると内容が頭に入らず、視聴離脱につながる

**修正依頼例：**
```
00:20 TE1（テロップタイミング修正お願いします）
```

---

## TE2｜自動文字起こし後の手動確認

**ルール：** 自動文字起こしを使った場合でも、必ず最初から最後まで手動で聞き直して確認する

**NG例：** 自動起こしのまま納品 → 「ChatGPT」が「チャットGPT」になっている

**OK例：** 聞き直しで誤変換を修正してから納品

**理由：** 固有名詞・専門用語は自動起こしが高確率で誤変換する。誤字は信頼感を損なう

**修正依頼例：**
```
05:49 TE2（誤変換確認お願いします）
```

---

## TE3｜テンプレートファイルの使用

**ルール：** テロップは必ず指定のmogrtファイルを使用する。フォントサイズ・色・スタイルを独自に変更しない

**NG例：** PremiereのデフォルトテキストでテロップをゼロFrom作成

**OK例：** `①通常時.mogrt`（1行160px）または`①通常時2行.mogrt`（2行130px）を使用

**理由：** チャンネル間で見た目を統一するため。独自変更が入ると修正工数が大幅に増える

**修正依頼例：**
```
00:10 TE3（指定テンプレ使用お願いします）
```

---

## TE4｜1行・2行テンプレートの使い分け

**ルール：** テロップが1行に収まる場合は1行用（160px）、収まらない場合のみ2行用（130px）を使う

**NG例：** 短いテロップなのに2行テンプレを使い、文字が小さくなっている

**OK例：** 「売れました」→ 1行用 / 「eBayのリミットが上がってきた」→ 2行用

**理由：** 文字が小さいと視認性が下がる。1行で収まるなら常に大きい方を使う

**修正依頼例：**
```
01:05 TE4（1行テンプレに変更お願いします）
```

---

## TE5｜テロップ最低表示時間

**ルール：** テロップの表示時間が1秒未満になる場合は、意訳して前後のテロップに統合するか、表示時間を延ばす

**NG例：** 「売れました」が0.5秒しか表示されず読めない

**OK例：** 前テロップと結合して「初めて利益20万円が出た瞬間です」として表示継続

**理由：** 読めないテロップはノイズになる。視聴者が読める最低時間は1秒

**修正依頼例：**
```
13:23 TE5（前テロップに統合お願いします）
```

---

## TE6｜演出テンプレートの使い分け

**ルール：** 発言の内容・トーンにあわせて以下のテンプレートを使い分ける

| 状況 | テンプレ | SE |
|------|---------|-----|
| 通常発言 | ①通常 | なし |
| 強調・ポイント | ①部分強調（黄色） | 強調系（ピーン） |
| タイトル・チャプター | ⑪タイトルテロップ | 迫力系（和太鼓） |
| 迫力・力強い発言 | ③④迫力テロップ | 迫力系 |
| ネガティブ・衝撃 | ⑮強ネガ | ネガ系（チリーン） |
| 豪華・成功 | ⑥キラテロップ | キラ系 |
| ボケ・軽いマイナス | ⑬よわよわテロップ | ボケ系（チーン） |
| ツッコミ | ⑭ツッコミ | ツッコミ系（バシ） |

**NG例：** 迫力ある発言なのに通常テロップ＋SEなし

**OK例：** 「月利100万達成！」→ 迫力テロップ＋スケールアップ＋迫力SE

**理由：** 演出とテロップのミスマッチは視聴者に違和感を与え、テンポが死ぬ

**修正依頼例：**
```
03:19 TE6（迫力テロップ使用お願いします）
```

---

## TE7｜テロップとSEは必ずセットで使う

**ルール：** SEを入れるときは必ず対応する演出テンプレートも使う。SE単体での使用はしない

**NG例：** 通常テロップのまま迫力SEだけ追加

**OK例：** 迫力テロップ＋スケールアップ＋迫力SE を同時に適用

**理由：** SEだけが鳴ると何を意味するか視聴者に伝わらず、むしろ混乱させる

**修正依頼例：**
```
00:38 TE7（演出とセットでお願いします）
```

---

## TE8｜SEとテロップの同時タイミング

**ルール：** SEはテロップ表示と同時に鳴らす

**NG例：** テロップが出てから0.3秒後にSEが鳴る

**OK例：** テロップ表示と同フレームでSEがスタート

**理由：** ズレると演出が間抜けに見える。視聴者の集中が一瞬切れる

**修正依頼例：**
```
00:38 TE8（SEタイミング修正お願いします）
```

---

## TE9｜テロップが画面の見せたい部分を隠さない

**ルール：** 商品・画面収録・グラフなど視聴者に見せたい要素の上にテロップを重ねない

**NG例：** 商品の値段が表示されている画面にテロップが重なって数字が読めない

**OK例：** テロップ位置を画面下部に調整、または1行に短縮して干渉を避ける

**理由：** 視聴者が見るべき情報が隠れると信頼感・満足度が下がる

**修正依頼例：**
```
07:16 TE9（テロップ位置調整お願いします）
```

---

## TE10｜複数話者の同時テロップ配置

**ルール：** 前の話者がまだ話している間に次の話者が話し始めた場合、前テロップを画面下に残したまま次テロップを上部に配置する

**NG例：** 前テロップを消してから次テロップを表示 → 発言が切れたように見える

**OK例：** 吉田さんのテロップ（下）を残しつつ、尾藤さんのテロップ（上）を追加表示

**理由：** 会話の流れが自然に見える。視聴者が誰の発言か混乱しない

**修正依頼例：**
```
26:13 TE10（前テロップ残してお願いします）
```

---

## TE11｜テロップの前後重なり・二重表示の禁止

**ルール：** 前のテロップが消える前に次のテロップが出ている状態（2枚重なり）にしない

**NG例：** 前テロップが消え切る前に次テロップが表示されている

**OK例：** 前テロップがアウト → 1フレームの間 → 次テロップがイン

**理由：** 重なりがあると文章として読めず、視聴者が混乱する

**修正依頼例：**
```
21:52 TE11（テロップ重なり修正お願いします）
```

---

## TE12｜ツッコミのテロップ位置

**ルール：** ツッコミ系テロップは演者の近くを基本としつつ、画面構成にあわせて位置を判断する

**NG例：** 画面下部に固定配置しており、演者や画面との関係が考慮されていない

**OK例：** 演者の近く、または画面の余白など見やすい位置に配置する

**理由：** 演者がツッコんでいる感を出す演出のため、画面下部への固定配置は避ける。ただし画面構成によって最適な位置は変わるため臨機応変に判断する

**修正依頼例：**
```
29:40 TE12（ツッコミテロップ位置見直しお願いします）
```

---

## TE13｜ボケ演出のテンプレート使用

**ルール：** 軽いマイナス・ボケシーンには⑬よわよわテロップ＋ボケ系SE（チーン）を使う

**NG例：** ボケシーンに通常テロップ＋SEなし

**OK例：** ボケているシーンにはよわよわテンプレ＋ボケSEを入れ、視聴者にわかるように演出する

**理由：** ボケの演出が入ることで視聴者がシーンの面白さを受け取りやすくなる。動画のエンタメ感を維持するためにも、ボケシーンは適切に演出する

**修正依頼例：**
```
29:40 TE13（ボケテロップ使用お願いします）
```

# TX｜意訳・テキスト

---

## TX1｜指示代名詞は具体語に置き換える

**ルール：** 「やつ」「これ」「それ」「あれ」「もの」などの指示代名詞は、何を指しているかが伝わる具体的な単語に置き換える

**NG例：** あれが売れた / それが問題で / こういうものが

**OK例：** R10のスリーブが売れた / 関税が問題で / こういう商品が

**理由：** 指示代名詞のまま出すと映像を見ていても意味が取れないことがある

**修正依頼例：**
```
00:35 TX1（指示代名詞を具体語に置き換えお願いします）
```

---

## TX2｜フィラー語・相槌はテロップにしない

**ルール：** 「えっと」「あの」「その」「ね」「なんか」「ちょっと」「やっぱし」「へ〜」「あ」「うん」などはテロップに起こさない。意図的に使う場合は除く

**NG例：** えっと、なんか、ちょっと → そのままテロップに起こしている

**OK例：** 発話の中のフィラー部分はテロップなし、次の意味のある発言からテロップ開始

**理由：** フィラー語をテロップにすると文字数が増えて読みにくくなり、内容が伝わりにくくなる

**修正依頼例：**
```
00:04 TX2（フィラー語のテロップ削除お願いします）
```

---

## TX3｜意味のない補助詞・語尾はテロップにしない

**ルール：** 「じゃないですか」「ですよね」「みたいな感じで」「〜なんですけど」など、文末についている意味の薄い語尾はテロップに起こさない

**NG例：** 売れるじゃないですか / 難しいですよね / そんな感じで

**OK例：** 売れる / 難しい / （テロップなし or 前テロップ継続）

**理由：** 語尾が多いとテロップが間延びして読みにくい。要点だけ残すことで視聴者の理解が速くなる

**修正依頼例：**
```
08:33 TX3（不要語尾のテロップ削除お願いします）
```

---

## TX4｜同じ意味の重複はどちらかに統一する

**ルール：** 演者が同じ意味を言い換えて繰り返した場合、テロップはどちらか一方に統一する

**NG例：** 「みなさん」と「ほとんどの方」を2行に分けてテロップに起こしている

**OK例：** 「みなさん」または「ほとんどの方」どちらかに統一して1テロップで表示

**理由：** 同じ意味のテロップが連続すると冗長になり、視聴者が混乱する

**修正依頼例：**
```
05:16 TX4（重複テロップをどちらかに統一お願いします）
```

---

## TX5｜文節単位で自然に区切る

**ルール：** テロップは日本語として意味のまとまりで区切る。文の途中で不自然に切らない

**NG例：**
- 「取材させていただく」「ときは昼以降は」「喫茶店に行きます」
- 「このリミットが」「埋まったらeBayに」「リミットアップ交渉をして」

**OK例：**
- 「取材させていただくときは」「昼以降は喫茶店に行きます」
- 「このリミットが埋まったら」「eBayにリミットアップ交渉をして」

**理由：** 不自然な区切りは文脈が掴めず、視聴者の読解ストレスになる

**修正依頼例：**
```
10:18 TX5（テロップの文節区切り修正お願いします）
```

---

## TX6｜改行位置を意味のまとまりで揃える

**ルール：** 2行テロップの改行は、意味のまとまりの切れ目で行う。単語の途中や助詞の直後で改行しない

**NG例：**
- 「利益が出せる\nようになりました」
- 「海外でもキティちゃん\nって今すごい」

**OK例：**
- 「利益が出せるように\nなりました」
- 「海外でもキティちゃんって\n今すごい」

**理由：** 改行位置がおかしいと一瞬読み止まりが起き、テンポが崩れる

**修正依頼例：**
```
18:19 TX6（改行位置修正お願いします）
```

---

## TX7｜テロップが短すぎる場合は前後と統合する

**ルール：** 1秒未満しか表示できない短い発言は、前後のテロップと意訳でつなげて1テロップにまとめる

**NG例：** 「そうですね」を単独でテロップ起こしし、0.5秒で消える

**OK例：** 「そうですね」はテロップなしにして前テロップをそのまま継続表示する

**理由：** 読めないテロップはノイズになる。前後でつなげた方が視聴者の理解がスムーズ

**修正依頼例：**
```
15:03 TX7（短すぎるテロップを前テロップ継続にお願いします）
```

---

## TX8｜カギ括弧「」はセリフ引用時のみ使う

**ルール：** 演者が他者のセリフや言葉を引用して話している箇所のテロップには「」をつける。通常の発言には不要

**NG例：** 通常の発言テロップに「」がついている / 引用発言なのに「」がない

**OK例：** 「分からないから買えないけど」（他者の言葉を引用している箇所）

**理由：** 「」があることで誰の発言かが明確になり、内容が伝わりやすくなる

**修正依頼例：**
```
09:18 TX8（セリフに「」をつけてください）
```

# HY｜表記統一

---

## HY1｜やわらかい語・補助動詞はひらがなで書く

**ルール：** 漢字でも書けるが、テロップでは読みやすさを優先してひらがなで統一する

| NG（漢字） | OK（ひらがな） |
|-----------|--------------|
| 出来る | できる |
| 色々・色んな | いろいろ・いろんな |
| お勧め | おすすめ |
| モノ・物 | もの |
| ご飯 | ごはん |
| 掛かって | かかって |

**NG例：** 出来る限り色々やってみて

**OK例：** できる限りいろいろやってみて

**理由：** 漢字が多いと視覚的に重くなり、企業チャンネルとして読みやすさが下がる

**修正依頼例：**
```
16:52 HY1（表記をひらがなに統一お願いします）
```

---

## HY2｜数値は単位まで明記する

**ルール：** 金額・数量は「円」「個」などの単位を省略しない

**NG例：** 10万 / 15万くらいまで利益いって

**OK例：** 10万円 / 利益15万円くらいまでいって

# SE｜効果音・演出

---

## SE1｜SEはシーンのトーン・発言内容に合わせて選ぶ

**ルール：** SEは発言の内容・雰囲気に合ったものを選ぶ。迫力系のセリフにシンバル音など、場の雰囲気と合わないSEは使わない

**NG例：** 力強い発言に場違いなシンバル音が入っている

**OK例：** 力強い発言には迫力系SE（和太鼓でドドン）、ネガシーンにはネガ系SE（チリーン）

**理由：** SEとテロップ内容がミスマッチだと視聴者が違和感を感じ、テンポが崩れる

**修正依頼例：**
```
22:40 SE1（SEがシーンと合っていません、差し替えお願いします）
```

---

## SE2｜同じSEを短い間隔で連続使用しない

**ルール：** 同じSEが続く場合は同種の別SEに変えて使い分ける

**NG例：** ボケシーンが連続しているのに同じ「チーン」が3回続く

**OK例：** 強調テロップ使用箇所は強調系SEの中から連続しないように使用している

**理由：** 同じSEの繰り返しは単調になり、演出効果が薄れる

**修正依頼例：**
```
10:15 SE2（同じSEが連続しています、使い分けお願いします）
```

---

## SE3｜SEはセリフの邪魔にならない音量に設定する

**ルール：** SEの音量はセリフが明瞭に聞こえる音量に抑える。SEがセリフと同じかそれ以上の音量にならない

**OK例：** セリフを基準音量として、SEをその下に収める

**理由：** SEが目立つと視聴者の集中がコンテンツから外れ、内容が伝わらなくなる

**修正依頼例：**
```
00:00 SE3（SEが大きいです、音量下げてお願いします）
```

# TR｜テンポ

---

## TR1｜明らかな間・言い直し・重複はカットする

**ルール：** 不自然に長い間・言い直し・同じ内容の繰り返しはカットしてテンポを保つ

**NG例：** 「えーと……（3秒の間）……そうですね」をそのまま残している

**OK例：** 無意味な間・言い直しをカットし、次の発言につなげている

**理由：** 不要な間が続くと視聴者の集中が途切れ、離脱につながる

**修正依頼例：**
```
05:10 TR1（不要な間・言い直しのカットお願いします）
```

---

## TR2｜カットが不自然な場合は倍速でつなぐ

**ルール：** カットすると映像がいきなり飛んで不自然になる場合は、カットせず該当部分を倍速にして自然につなぐ

**NG例：** 移動シーンをカットしたら映像が唐突に切り替わり、視聴者が状況を把握できない

**OK例：** 移動シーンを倍速にして場所移動の流れを残しつつテンポよくつないでいる

**理由：** 無理なカットは視聴者を混乱させる。流れを保ちながらテンポを上げる手段として倍速を使う

**修正依頼例：**
```
00:30 TR2（倍速でつないでお願いします）
```

---

## TR3｜台本に沿った発言はカットしすぎない

**ルール：** 台本に基づいた発言は基本的に残す。意味のある話・説明はカットせずに使う

**NG例：** 内容のある発言・説明をカットして話の流れが飛んでいる

**OK例：** 台本の流れを尊重し、内容に関わる発言は残してつないでいる

**理由：** ビジネス系チャンネルは情報の完結性が重要。カットしすぎると視聴者に伝わらない内容が出てくる

**修正依頼例：**
```
10:00 TR3（内容のある発言を残してお願いします）
```

---

## TR4｜リアクション・相槌はカットしない

**ルール：** インタビュー・対話形式の動画では、聞き手のリアクションや相槌はカットせずに残す

**NG例：** 「なるほど」「そうなんですね」などのリアクションをすべてカットし、会話が一方通行に見える

**OK例：** リアクションを残して会話のキャッチボール感を維持している

**理由：** リアクションがないと会話が不自然になり、視聴者が話の流れを追いにくくなる

**修正依頼例：**
```
01:00 TR4（リアクションを残してお願いします）
```

---

## TR5｜視覚変化を15〜20秒ごとに入れる

**ルール：** テロップの演出・画像・イラストなど何らかの視覚変化を15〜20秒ごとに入れる

**NG例：** 演者がしゃべるだけの映像が30秒以上続いている

**OK例：** 15〜20秒おきにポイントテロップ・図解・画像挿入などで視覚的な変化をつけている

**理由：** 視覚変化がないと単調になり視聴維持率が下がる。企業チャンネルでも視聴者を飽きさせない工夫が必要

**修正依頼例：**
```
08:00 TR5（視覚変化を入れてお願いします）
```

---

## TR6｜話が唐突に飛ぶ場合は前のシーンを活用する

**ルール：** 前後の文脈なしに突然話や場所が変わる場合は、つなぎとなるシーンを前後から探して使う

**NG例：** 店舗に到着したシーンで、到着前の移動や入口が映らず突然店内が映る

**OK例：** 到着前の移動・入口のシーンを挟み、視聴者が状況を把握してから店内映像に入る

**理由：** 脈絡のない場面転換は視聴者を混乱させ、動画の完成度が低く見える

**修正依頼例：**
```
28:49 TR6（前のシーンを使ってつないでお願いします）
```

**OK例：** 全体を通して音量が安定しており、繋ぎ目でも自然につながっている

**理由：** 音の問題は視聴者に最も強いストレスを与える。テロップや画角より先に気づかれる
