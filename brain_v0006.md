# FlarePocket Brain
**Version: v0006**
最終更新: 2026-04-01

---

## 1. プロジェクト概要

**FlarePocket** は日本語YouTube動画（主にビジネス系教育コンテンツ）のテロップ・演出を自動添削するシステム。

- 動画を1秒ごとにフレーム（JPEG）に分解し、AIがフレームを読み込みながらテロップ等の問題を指摘する
- チェック対象：誤字・固有名詞表記・口語表現・全角数字・改行位置・モザイク漏れ・バナー・演出
- 出力：`output/backlog/{動画タイトル}/review_report.txt`

## 2. チェックルール（カテゴリ一覧）

| コード | カテゴリ | 内容 |
|--------|----------|------|
| **HY1** | 誤字・固有名詞 | 誤字脱字、固有名詞の誤表記、ひらがな優先語・数値単位・概数・長音符・スペース（下記詳細参照） |
| **HY2** | 口語・語彙ミス | 二重否定、二重敬語、口語、助詞抜け、ら抜き言葉、指示代名詞、演者の口語・誤発言（下記詳細参照） |
| **HY4** | 全角数字→半角 | テロップ内の全角数字（１２３など）を半角に修正 |
| **HY6** | 改行・短テロップ | 助詞直後改行、語の途中改行、文節を無視した不自然な区切り、短すぎる1行テロップ |
| **HY7** | 行頭「って」 | 行頭・テロップ冒頭が「って」で始まる場合は「という」に修正 |
| **MO** | モザイク漏れ | ASIN/セラーID/仕入価格/個人情報/一般人の顔の露出 |
| **SE** | 演出テロップ | 強調・迫力発言に通常テロップのまま（演出テンプレ未使用）、ネガシーンに演出なし |
| **CA** | バナー・ロゴ・画角 | バナー未表示、ロゴ欠け、エンドカード誤字、強調シーンのスケールアップなし、ネガシーンのスケールダウンなし |
| **TX2** | カット候補 | 相槌のみ（「うん」「なるほど」単独）、話途中カット |
| **TX3** | 不要語尾・フィラー語 | 「えっと」「なんか」「じゃないですか」「みたいな感じで」等のテロップ起こし |
| **TX4** | 重複テロップ | 同じ意味の言い換えが2テロップ連続→どちらかに統一 |
| **TX5** | 文節区切りミス | テロップが意味のまとまりを無視して不自然に分割されている |
| **TX8** | カギ括弧誤用 | 「」は他者のセリフ引用時のみ使用。通常発言への付加・引用への欠落を指摘 |

### HY1 詳細ルール

**ひらがな優先語（漢字→ひらがな）**
| NG | OK |
|----|-----|
| 出来る | できる |
| 良い | いい |
| 物・モノ | もの |
| ご飯 | ごはん |
| 色々・色んな | いろいろ・いろんな |
| お勧め・オススメ | おすすめ |
| 掛かる | かかる |
| 位 | くらい |

**数値・単位ルール**
- 金額・数量は単位を省略しない（「10万」→「10万円」、「15個」など）
- 概数は正しい日本語表記で（「数100円」→「数百円」、「累計8万円」→「累計約8万円」）

**カタカナ語の語尾長音符**
- 語尾の「ー」は原則省く（「データー」→「データ」、「プリンター」→「プリンタ」、「コンピューター」→「コンピュータ」）

**テロップ内の不要スペース**
- 半角・全角スペースの混入を削除（「Chat GPT」→「ChatGPT」、「eBay　で」→「eBayで」）

### HY2 詳細ルール

**指示代名詞は具体語に置き換える**
- 「やつ」「これ」「それ」「あれ」「もの」等が何を指すか明確でない場合→具体的な名称に修正
- 例：「あれが売れた」→「R10のスリーブが売れた」

**演者の口語・誤発言も正しい表記に直す**
- 例：「深ぼっていければ」→「深掘りしていければ」
- 例：「あんまり」→「あまり」、「ねぇな」→「ないな」

### 変換ミスチェックリスト（henkanMissList.md）
- `henkanMissList.md` に載っている変換ミスは**参考リスト**として活用する
- manualと異なり**厳守ではない**。同じ文面があっても前後の文脈を見て判断する
- 似た文章・表現があった場合は精査してから指摘するかどうか決める
- 文脈上自然・意図的な用法と判断できる場合は指摘しない

### 固有名詞・正式表記チェックリスト
- YouTube（大文字T）
- Amazon（「アマゾン」不可）
- Payoneer（「ペイオニア」不可）
- Yahoo!ショッピング（「！」込みが正式）
- S&P500（500まで含める）
- WEBセミナー（「ウェブセミナー」不可）
- eBay（「ebay」不可）/ Shopee（「Shoppy」「ショップ輸出」不可）/ SHOPEE / Sekai Pocket / Zoom / SLS / BUYMA / ASIN
- ChatGPT（「Chat GPT」「チャットGPT」「GPT」不可）
- UPS（「UBS」不可）
- 物販総合研究所（「物総研」不可）
- バレエ（「バレー」は誤り）
- スムーズ（「スムース」は誤り）
- アベニュー・モンテーニュ（「アベニモンターニュ」は誤り）

### MOルールの例外（指摘不要）
- バーコード・商品価格 → 指摘不要（編集D確認済み）
- 実践者の顔写真 → 指摘不要（同意取得済み前提）
- 受講生の成果数字（実名なし）→ 基本指摘不要

## 3. 添削方針

### 厳しさ
**「疑わしければ指摘する」** 方針。指摘数が多くなって構わない。見逃しの方が問題。

- 迷ったら指摘する（省かない）
- 誤字・表記統一・改行位置・1行テロップは量が多く出て当然
- 「演出が足りないかも」「スケールが小さいかも」なども積極的に出す

### 出力形式（必須・この形式以外は不可）
```
- MM:SS HY1 テロップ「xxx」→「yyy」に修正
- MM:SS HY2 テロップ「xxx」→口語表現。「yyy」に修正
- MM:SS CA バナーが未表示。確認してください
```

- 時刻は `MM:SS` または `H:MM:SS`
- 表（テーブル）は使わない、箇条書きのみ
- カテゴリコードを必ず付ける
- HY6（改行位置）の指摘は「意味の切れ目で修正」等の曖昧表現は不可。修正後のテロップ全文を `→「1行目／2行目」` の形式で必ず示す

## 4. 絶対に守るルール（幻覚防止・タイムコード精度）

### ① フレームを見ずに指摘しない
- 「このシーンにこのテロップがあるはず」と推測して指摘しない
- 指摘前に必ず `read_frame` で該当フレームを目視確認する
- フレームを確認して「確かに存在する」と確認できた内容のみ指摘する
- 疑わしい場合は前後3〜5フレームも確認してから判断する

### ② タイムコードは計算・推測しない
- `read_frame` が返したファイル名の時刻をそのまま使う
  - 例：`frame_01346.jpg` → タイムコードは `22:26`（1346÷60=22余26）
- 自分でフレーム番号から時刻を計算して推測しない
- テロップが最初に出現するフレームを特定してから記載する

### ③ 疑問文「？」は必ずフレームで確認
- テロップに「？」がないと思っても、実際のフレームを開いて目視確認してから指摘する
- script.txt（自動生成字幕）は精度が低く「？」なしに見えても実際のテロップには付いていることが多い

### ④ スライド上のテロップ・バナーは指摘不要
- スライドが表示されているシーンで、テロップやバナーが重なるのは正常な編集スタイル
- スライド上のバナー重なりは指摘しない

### ⑤ 句読点は指摘しない
- テロップに読点（、）や句点（。）がないのは**このチャンネルの正常スタイル**
- 「読点欠落」「句点なし」の指摘は出さない
- テロップに句読点が**入っている**場合（「。」「、」が不要に含まれている）は指摘する

### ⑥ チャンネル固有の用語は誤字扱いしない
- 「デカ重」など、チャンネルで使われている造語・スラングは誤字として指摘しない
- 不明な場合は「造語の可能性あり。演出意図の確認推奨」と添え書きする

## 5. 見逃しやすい項目（重点確認リスト）

### 最重要：見逃しが多いカテゴリ
1. **表記統一** — 「バイマ→BUYMA」のような揺れ。全フレームで一字一句確認
2. **語彙ミス** — 「バレー→バレエ」「スムース→スムーズ」など意味的な誤り
3. **数字範囲フォーマット** — 「40~50万円」→「40万~50万円」（万などの単位は各数字の後ろに付ける。円などの通貨単位は末尾1箇所のみでOK）
4. **助詞抜け** — 「クラファンオンラインで」→「クラファンをオンラインで」
5. **冗長語** — 「とかってなったら」「辞めてやめていくんで」「あと2つ目」
6. **送り仮名統一** — 「売上」vs「売り上げ」が動画内で混在していないか
7. **テロップ見切れ** — 画面端で文字が切れていないか（長文・英語・計算式は特に注意）
8. **テロップへの不要記号** — 「w」「ｗ」「！」「…」「。」「、」が不適切に入っていないか
9. **ひらがな優先語** — 「良い→いい」「位→くらい」「ご飯→ごはん」「出来る→できる」「お勧め/オススメ→おすすめ」
10. **話途中カット** — 字幕が「〜ので」「〜て」「〜ですが」等で終わっているシーン
11. **数値の単位省略** — 「10万」→「10万円」のように単位が抜けていないか
12. **カタカナ語の語尾長音符** — 「データー」→「データ」「プリンター」→「プリンタ」
13. **不要スペース** — テロップ内の半角・全角スペース混入（「Chat GPT」→「ChatGPT」）
14. **指示代名詞** — 「やつ」「これ」「それ」「あれ」が何を指すか不明なまま→具体名に要修正
15. **TX3フィラー語** — 「えっと」「なんか」「じゃないですか」「みたいな感じで」がテロップに残っていないか
16. **TX4重複テロップ** — 同じ意味の言い換えが2テロップ連続していないか

### 演出系の見逃し
- 演出シーンでバナーが非表示になっているか（先出しハイライト・迫力演出時は非表示が基本）
- 章タイトル・ポイント発表シーンでSE・演出テロップがあるか（通常テロップのままでないか）
- CTAセクションでQRバナーが毎回表示されているか（後半の再告知も含む）
- 複数箇条書きシーンで1項目ずつ小出しになっているか
- 強調・力強い発言シーンでスケールアップ（ズーム）が入っているか
- ネガティブ・衝撃シーンでスケールダウン・モノクロなどの暗め演出があるか

### 演出テンプレート使い分け（SEカテゴリ判定基準）
| シーン | 期待される演出 |
|--------|---------------|
| 通常発言 | ①通常テロップ＋SEなし |
| 強調・ポイント発言 | 部分強調テロップ（黄色）＋強調SE |
| タイトル・チャプター | タイトルテロップ＋迫力SE（和太鼓） |
| 迫力・力強い発言 | 迫力テロップ＋スケールアップ＋迫力SE |
| ネガティブ・衝撃 | 強ネガテロップ＋スケールダウン＋ネガSE（チリーン） |
| 豪華・成功 | キラテロップ＋キラSE |
| ボケ・軽いマイナス | よわよわテロップ＋ボケSE（チーン） |
| ツッコミ | ツッコミテロップ（演者近く）＋ツッコミSE（バシ） |

→ フレームで「強調・ポイント発言なのに通常テロップのまま」「迫力発言なのにSE・演出なし」等を確認したらSEとして指摘

### フレームだけでは確認不可（推測として指摘 or 「確認推奨」と明記）
- BGM音量・SE音量
- テロップ表示タイミングのズレ（TE1）
- テロップの前後重なり・二重表示（TE11）
- 色調・コントラスト（JPG圧縮で判断不可）
- 右上ロゴの素材種類
- テロップのスケール適正

## 6. ワークフロー

### Step 1: フレーム抽出
```bash
ffmpeg -i input.mp4 -vf fps=1 -q:v 2 frames/frame_%05d.jpg
```
- 1秒ごと1フレーム（fps=1）が基本。変更不可
- フレーム命名規則：`frame_00001.jpg`〜`frame_NNNNN.jpg`
- タイムコード変換：`frame_01346.jpg` → `1346 // 60 = 22分`, `1346 % 60 = 26秒` → `22:26`

### Step 2: 並列レビュー（3セクション分割）
- 動画を3等分して3つのエージェントが同時に担当
- 各エージェントは3〜5フレーム間隔でスキャン（これ以上広げると漏れが出る）
- 疑わしい箇所は前後フレームも確認

セクション分割例（1562フレームの場合）：
- Section 1: frame 1〜520（00:00〜08:40）
- Section 2: frame 521〜1041（08:41〜17:21）
- Section 3: frame 1042〜1562（17:22〜26:02）

### Step 3: script.txt の扱い
- `input/script.txt`（自動生成字幕）は参考程度
- 全文精読しない。固有名詞・ブランド名・通貨などをgrepで抽出するのみ
- script.txtは精度が低く信用できないため、フレーム目視が主たる判断根拠

### Step 4: レポート出力
```
output/backlog/{動画タイトル}/review_report.txt
```

フォーマット：
```
# {動画タイトル} 添削レポート

指摘件数: N 件

## HY1（誤字・固有名詞）
- 00:34 HY1 テロップ「xxx」→「yyy」に修正

## HY2（口語表現・語彙ミス）
- 03:29 HY2 テロップ「xxx」→「yyy」に修正

...
```

### Step 5: ファイル削除
添削完了後は必ず実施：
1. 動画ファイル（`tmp/{job_id}/input.mp4`）を削除
2. フレーム（`tmp/{job_id}/frames/`）を削除
- レポートのみ残す（動画1本あたり約154MB節約）

### Step 6: フィードバックの記録
ユーザーから誤検出・見逃しの指摘があった場合、および**添削以外の指示・ルール変更・フィードバックを受けた場合も**、`learnings/{REVIEWER_NAME}_learnings.md` に追記する。
- 「learningsに反映して」と言われなくても**自動で**記録する
- 記録内容はそのままの事実ではなく「次回以降に活かせる学習ルール」の形で書く
  - NG：「17:22が誤りだった」
  - OK：「タイムコードは必ずフレームファイル名から取得し計算で出さない」

```markdown
## 2026-03-27
- チャンネル固有の造語（「デカ重」等）は誤字扱いしない
- 「借して」→「貸して」の誤用は見逃しやすいため注意（HY1）
- タイムコードは必ずフレームファイル名から取得する。計算で出すとズレる
```

- `learnings/` フォルダが存在しない場合は自動作成する
- `.env` の `REVIEWER_NAME` が名前になる（未設定の場合は `unknown`）
- このファイルはfushimiが定期的に収集し、brainへの反映可否を判断する

### Step 2 補足: 並列エージェントへのルール参照
- 添削エージェントには **brain.md（全共通ルール）** と **learnings/{REVIEWER_NAME}_learnings.md（担当者固有の最新ルール）** を両方渡す
- **brainが優先**。learningsがbrainと矛盾する場合はbrainのルールに従う
- learningsはbrainに未反映の個人FBが即時反映されるための補助的な参照

### Step 7: learningsのbrain格上げ確認（fushimi専用）
fushimiが「learningsを確認したい」と依頼した場合：
1. `learnings/` フォルダ内の全ファイルを一覧表示
2. 各ファイルの内容を読み、以下を報告する
   - **brain未反映のルール一覧**（格上げ候補）
   - **brainと重複している項目**（learningsから削除してよい候補）
   - **brainと矛盾している項目**（どちらが正しいかfushimiに確認）

### Step 8: Brainバージョン管理（fushimi専用）
fushimiが「反映して」と指示した時のみ実行。以下の順で行う：
1. `FlarePocket_brain.md` の Version を +1・最終更新日を更新
2. `FlarePocketManual/brain_v000X.md` としてコピー
3. `FlarePocketManual/index.html` のダウンロードリンク・バージョンバッジ・最終更新日を更新
4. `FlarePocketManual/` で `git add → git commit → git push`（リポジトリ: TatsuyaFushimi/FlarePocketManualPage）

### Step 9: セットアップ要件変更時の対応（fushimi専用）
`requirements_web.txt` または `.env` の必要項目・必要ツールに変更があった場合：
- `FlarePocketManual/index.html` の STEP 1 セクションを自動で更新する

### Step 10: 作業後の案内文について
- セットアップ手順は `FlarePocketManual/index.html` に記載済みのため、作業完了後に「〜すれば使い始められます」等の案内を添えない

## 7. スライド対応

スライドへの指摘が出た場合は、まずユーザーに確認する：
> 「このスライドは編集可能ですか？」

編集不可のスライドへの修正指摘は意味がないため、確認なしに指摘を出さない。

## 8. エンジン選択

### Claude（デフォルト・推奨）
- フレームを1枚ずつ読み込みながら添削（`read_frame` ツール使用）
- タイムコードはフレームファイル名から自動取得
- 4並列エージェントで処理
- ANTHROPIC_API_KEY が必要（Claude Code VSCode拡張経由で自動注入）

### Gemini（オプション・要明示指示）
- **ユーザーが「Gemini」と明示指示した場合のみ使用する。指示がない場合は必ずClaudeエンジンで処理する。**
- 動画ファイルをGemini File APIに直接アップロードして添削
- 55分以上の動画は前半/後半に分割 → 後半タイムコードにオフセット加算
- GEMINI_API_KEY が `.env` に必要
- モデル：`gemini-2.5-flash`（2M tokenコンテキスト版が必要な場合は `gemini-2.5-pro`）

## 9. よくある誤検出パターン（出さないこと）

| 誤検出パターン | 理由 |
|----------------|------|
| 読点（、）欠落 | このチャンネルは句読点なしが正常スタイル |
| バーコードMO指摘 | 指摘不要（編集D確認済み） |
| 実践者顔写真MO指摘 | 同意取得済み前提で指摘不要 |
| スライド上テロップ重なり | 正常な演出スタイル |
| 冒頭SEなし指摘 | フレームからは判定不可 |
| 「デカ重」等チャンネル固有語 | 造語・スラングは誤字扱いしない |
| フレーム未確認の疑問文「？」指摘 | script.txt精度が低く誤検出多い |
| 「より丁寧な表現に修正」系（具体的修正案なし） | 具体的な`→yyy`がない指摘は出さない |
